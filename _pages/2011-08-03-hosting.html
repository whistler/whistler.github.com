Free Hosting There are a few free Rails hosts but you probably would not go pretty far with that. However Heroku is a nice place which hosts small Rails applications for free but its performance is not ideal. Shared Hosting You might be considering getting a shared hosting. There are many available that provide Rails support some of the good ones are: Site5, Bluehost & Dreamhost. However a shared environment is not well suited for Rails applications due to their heavy memory requirements. The ideal solution would be to get your own VPS. VPS Hosting If money is not an issue there are plenty of good VPS hosts around, but if your website would not immediately get 1000+ visitors per day, you might want to consider getting a low-cost VPS. Dont expect truely amazing performance from these VPS hosts, try not to get the cheapest one as it might be bad. But they are good enough to host some simple applications. These servers are unmanaged which means you have to configure your server yourself but this also gives you the flexibility to configure your server exactly like you want. How much memory do you need One question you would ask when buying a VPS is how much memory is enough for your Rails app? This depends on your application and server configuration. 128 MB should be enough if you are running a small lightweight application on SQLite. If you are planning on running MySQL, it takes a lot of memory and you should probably go for atleast 512MB, although 256 might be doable if you have a light configuration. Self-hosting at home It might be a good idea to host from home if you have an unlimited Internet connection and an old computer laying around. But be careful about setting up your router and firewall right to keep yourself secure. Configuring your server Ok, so now you got your VPS host or your old computer you want to host on, how do you set up the server.
